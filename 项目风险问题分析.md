## P0：管理员鉴权与授权

### 当前状态
**配置型 RBAC 已实现。**

后端使用环境变量：

```powershell
$env:ADMIN_ACCOUNTS="editor|editor-token|KNOWLEDGE_READ,KNOWLEDGE_WRITE;reader|reader-token|KNOWLEDGE_READ"
```

账户格式：

```text
username|token|ROLE_1,ROLE_2
```

支持角色：

- `ADMIN`：全部管理员权限
- `KNOWLEDGE_READ`：允许 GET、HEAD
- `KNOWLEDGE_WRITE`：允许 POST、PUT、PATCH、DELETE

前端继续使用：

```text
VITE_ADMIN_API_TOKEN=<对应账户的 token>
```

鉴权结果：

| 场景 | 状态码 |
| --- | ---: |
| 令牌缺失或错误 | `401` |
| 账户缺少操作权限 | `403` |
| 后端没有任何管理员账户配置 | `503` |
| 权限满足 | 正常执行 |

为兼容旧环境，`ADMIN_API_TOKEN` 仍可作为一个拥有 `ADMIN` 角色的旧式账户使用；新环境应优先配置 `ADMIN_ACCOUNTS`。

### 审计

管理员日志记录：

- 请求方法
- 请求路径
- HTTP 状态码
- 来源地址
- 账户名
- 角色
- success/error

不会记录 token 原文。权限不足时也会记录已认证账户，例如只读账户尝试写操作会记录 `actor=reader` 和 `status=403`。

### 剩余风险

当前账户和角色来自环境变量，不是数据库用户中心，尚不具备：

- 登录会话
- token 过期和吊销
- 管理员账户在线管理
- 持久化审计查询

如果未来正式公网部署，仍建议迁移到 Spring Security、数据库用户/角色表、短期访问令牌和持久化审计表。
