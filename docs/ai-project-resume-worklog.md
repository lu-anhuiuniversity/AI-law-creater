## 项目定位

本项目面向劳动合同法律咨询场景，拟构建一个基于大模型与 RAG 的智能问答系统。系统目标是围绕劳动合同签订、试用期、工资拖欠、加班工资、未签合同、经济补偿、劳动仲裁、合同条款风险等高频问题，提供结构化回答、法律依据引用、维权流程指引和证据清单。

系统定位为法律信息检索与咨询辅助工具，不替代律师、仲裁机构或法院意见。项目强调可追溯、可解释、可演示和学生团队可实现。

### 8. CORS 跨域问题分析与解决

- 老师或队友本地运行时，常见问题是前端 `localhost:5173` 请求后端 `localhost:8080` 触发浏览器 CORS 限制。
- 本项目采用双层解决方案：
  - 前端 Vite 代理：`/api` 转发到 `http://localhost:8080`，开发阶段避免浏览器跨域。
  - 后端 CORS 配置：`WebMvcConfig` 允许 `localhost:5173`、`127.0.0.1:5173`、`localhost:4173`、`127.0.0.1:4173` 访问 `/api/**`。
- 额外记录了端口占用问题：
  - 验证时发现 `8080` 已被占用。
  - 未直接停止用户进程，而是使用临时端口 `8081` 验证新 CORS 配置。
  - CORS 预检返回 `Access-Control-Allow-Origin: http://localhost:5173`，说明后端跨域配置生效。

可写入简历的表达：

- 解决前后端分离开发中的跨域问题，通过 Vite dev proxy 与 Spring Boot CORS 配置实现本地开发联调，补充端口占用排查和运行文档，提升小组成员与评审环境中的可运行性。

## 项目能力点总结

### 产品与需求分析

- 明确项目边界：法律信息检索与咨询辅助，不替代律师服务。
- 将复杂法律咨询场景拆解为智能问答、依据引用、维权流程、证据清单和合同审查模块。
- 为学生实训场景控制 MVP 范围，优先保障可实现性和演示完整度。

### AI 与 RAG 思维

- 采用 RAG 思路降低大模型幻觉风险。
- 强调回答必须基于法规、司法解释、人社政策文件或项目知识库材料。
- 设计“依据不足”与“安全拒答”机制，避免无来源的确定性法律结论。

### 前端设计

- 规划法律类 AI 产品的信息架构和交互路径。
- 设计三栏式咨询工作台：问题上下文、结构化回答、引用依据。
- 关注法律场景特有 UI 问题：长标题、条款引用、来源权威性、时效状态、风险等级和免责声明。

## 面试可讲要点

- 为什么选择劳动合同作为法律 AI 场景：
  - 高频、问题边界相对清晰、法规依据集中、适合学生团队做 MVP。
- 为什么选择 RAG：
  - 法律场景不能只依赖模型生成，需要可追溯来源来降低幻觉和误导风险。
- UI 设计最重要的点：
  - 用户要先看懂结论，但必须能快速核对依据和下一步行动。
- 项目边界：
  - 系统提供法律信息检索与辅助咨询，不提供正式法律意见，不承诺仲裁或诉讼结果。
- 后续可量化指标：
  - 常见问题命中率
  - 引用来源覆盖率
  - 问答响应时间
  - 用户反馈评分
  - 知识库文档数量和片段数量
