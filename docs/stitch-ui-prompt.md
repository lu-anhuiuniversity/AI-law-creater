# Stitch UI 设计提示词：劳动合同法律助手

请为一个中文 Web 产品生成高保真 UI 设计，产品名称为「劳动合同法律助手」。

## 产品定位

这是一个基于大模型与 RAG 的劳动合同智能咨询系统，用于学生实训项目、法律咨询原型和劳动合同问题智能问答。系统帮助普通劳动者、应届毕业生/实习生、企业 HR、小微企业负责人快速理解劳动合同相关问题，并提供可追溯法律依据、维权流程和证据清单。

产品不是律师服务，不替代律师、仲裁机构或法院意见。界面必须清楚展示免责声明和风险提示。

## 设计参考

参考但不要照抄以下成熟法律产品的信息架构和视觉气质：

- Westlaw：专业法律研究工具，强调可信来源、引用状态、法规版本对比、研究效率。
- Lexis+ with Protégé：法律 AI 产品，强调权威来源、可追溯回答、研究/起草/分析一体化。
- Justia：面向公众的法律信息门户，分类清晰、低门槛、适合普通用户浏览。
- 国家法律法规数据库：中国法律法规检索产品，强调来源权威、条文检索、文档结构。
- 北大法宝/威科先行：中文法律检索产品，可参考其法规、案例、专题检索的严谨感。

整体方向：专业、可信、克制、清晰、可操作。不要做夸张营销页，不要做炫酷 AI 科技风首页，不要大面积紫色渐变，不要使用抽象法律锤子插画作为核心视觉。第一屏就是可用的咨询工作台。

## 目标用户

- 普通劳动者：想知道加班工资、试用期、未签合同、工资拖欠、离职补偿等问题。
- 应届毕业生/实习生：需要低门槛解释三方协议、试用期、劳动合同、社保公积金。
- 企业 HR 或小微企业负责人：希望快速核对劳动合同条款和用工风险。
- 项目评审者：关注功能完整度、数据来源可信度、展示效果和实现可行性。

## 页面范围

请生成一套完整 Web App UI，包含桌面端主设计，并考虑移动端响应式状态。至少包含以下页面或主要视图：

1. 咨询工作台首页
2. 智能问答结果页
3. 法律依据展开面板
4. 维权流程与证据清单视图
5. 合同条款审查页
6. 知识库管理后台
7. 问答历史与反馈页

## 信息架构

左侧为固定导航栏，顶部为项目状态栏，主区域为工作台。

导航项：

- 智能咨询
- 合同审查
- 法律依据库
- 问答历史
- 反馈处理
- 知识库管理
- 系统说明

顶部状态栏展示：

- 当前知识库版本
- 最近更新时间
- 已启用法规/政策文件数量
- 免责声明入口
- 用户角色切换：普通用户 / 管理员演示

## 视觉风格

配色：

- 主色：深法蓝 `#16324F`
- 辅助色：墨绿 `#246B5A`
- 强调色：琥珀金 `#B7791F`
- 背景：浅灰白 `#F6F7F9`
- 文本主色：近黑 `#1F2933`
- 风险色：红棕 `#B42318`
- 成功色：稳重绿 `#2F855A`

字体：

- 中文优先使用系统无衬线字体，例如 PingFang SC、Microsoft YaHei、Noto Sans SC。
- 标题使用较重字重，正文强调可读性。
- 法律依据、条款编号、引用来源可以使用等宽或紧凑数字样式增强检索感。

布局：

- 管理工具风格，信息密度中等偏高，但不要拥挤。
- 卡片圆角不超过 8px。
- 使用清晰分栏：问题输入区、回答区、依据区、流程区。
- 多使用标签、表格、折叠面板、步骤条、引用卡片，而不是营销式大卡片堆叠。

## 核心页面设计要求

### 1. 咨询工作台首页

页面中心是大号咨询输入框，占据第一视觉焦点。

输入框 placeholder：

「例如：入职两个月没签劳动合同，我能要求赔偿吗？」

输入框下方提供问题类型快捷入口：

- 未签劳动合同
- 试用期解除
- 加班工资
- 工资拖欠
- 经济补偿
- 社保公积金
- 劳动仲裁
- 合同条款风险

右侧显示「咨询边界」提示：

- 本系统提供劳动合同法律信息检索与咨询辅助
- 回答需结合证据、地区政策和实际情况判断
- 重大争议建议咨询律师或当地劳动保障部门

首页还应展示三个小组件：

- 高频问题
- 最近更新法规
- 知识库可信度概览

### 2. 智能问答结果页

结果页采用三栏布局：

- 左侧：本次问题、问题分类、追问历史
- 中间：结构化回答
- 右侧：引用依据、风险提示、可执行操作

回答必须分为以下模块：

1. 结论摘要
2. 法律依据
3. 维权流程
4. 证据清单
5. 注意事项

每条关键结论旁边显示引用角标，例如 `[依据 1]`、`[依据 2]`。点击角标后，右侧依据面板滚动到对应来源。

回答顶部显示：

- 问题类型：未签劳动合同
- 风险等级：中
- 回答可信度：依据充分 / 依据不足
- 生成时间

回答底部操作：

- 继续追问
- 复制答案
- 保存会话
- 导出为文本
- 反馈有误

### 3. 法律依据展开面板

依据面板以卡片列表展示来源：

每张来源卡片包含：

- 来源名称：例如《中华人民共和国劳动合同法》
- 条款：例如第十条、第八十二条
- 发布/生效信息
- 来源类型：法律法规 / 司法解释 / 人社政策 / FAQ
- 相关片段摘要
- 权威性标签
- 展开原文

展开后显示命中片段，并高亮与用户问题相关的关键词。

当依据不足时，用明显但克制的警示样式提示：

「当前知识库没有找到足够直接的依据，以下回答仅作一般信息参考。」

### 4. 维权流程与证据清单视图

用步骤条展示行动路径：

1. 与用人单位沟通
2. 保存证据
3. 向劳动保障监察部门投诉
4. 申请劳动仲裁
5. 必要时提起诉讼

每一步包含：

- 目标
- 准备材料
- 注意事项
- 是否适合当前问题

证据清单使用 checklist 设计：

- 劳动合同或录用通知
- 工资流水
- 考勤记录
- 加班通知
- 聊天记录
- 工作成果
- 社保缴纳记录
- 离职证明或解除通知

### 5. 合同条款审查页

左侧为合同条款输入区，支持粘贴文本。

右侧为审查结果：

- 总体风险等级
- 风险条款列表
- 风险原因
- 可能涉及依据
- 修改建议

审查类型标签：

- 试用期
- 工资工时
- 解除终止
- 保密竞业
- 社保福利

示例输入提示：

「粘贴劳动合同中的试用期、工资、解除、竞业限制等条款，系统将辅助识别明显风险点。」

### 6. 知识库管理后台

面向管理员演示，布局更偏数据管理。

功能区：

- 上传法规/政策/FAQ 文件
- 文档解析状态
- 向量索引状态
- 文档启用/停用
- 版本管理
- 元数据编辑

表格字段：

- 文档标题
- 来源类型
- 发布日期
- 适用范围
- 版本状态
- 切分片段数
- 索引状态
- 最近更新
- 操作

顶部数据卡：

- 已入库文档数
- 可检索片段数
- 引用覆盖率
- 低评分问答数

### 7. 问答历史与反馈页

问答历史列表展示：

- 问题标题
- 问题分类
- 最近追问时间
- 是否已保存
- 回答反馈状态

反馈管理展示：

- 点赞/点踩
- 错误类型：依据错误、流程不清、超出范围、回答不完整
- 管理员处理状态：待处理、已修正、已忽略

## 关键交互

- 用户输入问题后，界面进入“检索依据 → 生成回答 → 展示来源”的过程状态。
- 生成中不要只显示转圈，要显示阶段进度：
  - 正在识别问题类型
  - 正在检索劳动法规和政策文件
  - 正在生成结构化回答
  - 正在校验引用来源
- 所有 AI 回答都必须可回到来源。
- 超出范围问题要展示安全拒答状态，例如：
  「这个问题超出劳动合同咨询范围，建议咨询专业律师或当地主管部门。」
- 用户输入身份证号、手机号、详细住址等敏感信息时，界面提示脱敏。

## 文案基调

专业、克制、明确，不要过度营销。

示例文案：

- 「先看结论，再核对依据」
- 「以下内容为法律信息检索结果，不构成正式法律意见」
- 「依据不足，建议补充合同、工资流水或解除通知等材料」
- 「可执行下一步」
- 「保留证据后再进行投诉或仲裁」

## 组件要求

请设计以下组件：

- 问题输入框
- 问题类型标签
- 结构化回答卡片
- 来源引用卡片
- 法律条文展开面板
- 维权步骤条
- 证据 checklist
- 风险等级 badge
- 合同条款审查结果卡
- 知识库文档表格
- 反馈状态标签
- 免责声明提示条
- 空状态、加载状态、错误状态

## 响应式要求

桌面端：

- 优先三栏工作台布局。
- 左侧导航固定，中间内容可滚动，右侧依据面板可折叠。

移动端：

- 底部导航替代侧边栏。
- 回答、依据、流程使用 tabs 切换。
- 问题输入框固定在底部或顶部显著位置。

## 必须避免

- 不要做律师事务所官网风格的营销首页。
- 不要使用大幅法槌、天平、法院柱子作为唯一视觉元素。
- 不要让 AI 回答看起来像没有来源的聊天机器人。
- 不要隐藏免责声明。
- 不要使用过度活泼、游戏化或娱乐化风格。
- 不要把法律依据放在很深的二级页面里。

## 输出要求

请输出一套高保真 UI，包括：

- 桌面端主工作台
- 智能问答结果页
- 合同审查页
- 知识库管理后台
- 移动端关键页面预览
- 明确的设计系统：颜色、字体、组件状态、间距、图标风格

设计应让评审者一眼看出：这是一个可信、可追溯、可演示的劳动合同 RAG 法律助手。
