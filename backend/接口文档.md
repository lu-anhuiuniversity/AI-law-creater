# 后端接口文档

## 基础信息

默认服务地址：

```text
http://127.0.0.1:8080
```

如果启动时指定了备用端口，例如 `--server.port=18083`，请将文档中的 `8080` 替换为实际端口。

## 统一返回格式

后端接口统一返回 `ApiResponse<T>` 格式：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `code` | `number` | 业务状态码，`0` 表示成功，非 `0` 表示失败。 |
| `message` | `string` | 请求处理结果说明。 |
| `data` | `object` / `null` | 实际响应数据，不同接口的数据结构不同。 |
## 管理员接口鉴权

以下接口位于 `/api/admin/**`，必须携带管理员令牌：

```http
X-Admin-Token: <account-token>
```

后端通过环境变量配置账户和角色：

```powershell
$env:ADMIN_ACCOUNTS="editor|replace-with-editor-token|KNOWLEDGE_READ,KNOWLEDGE_WRITE;reader|replace-with-reader-token|KNOWLEDGE_READ"
```

每条账户配置格式为：

```text
username|token|ROLE_1,ROLE_2
```

当前角色：

- `ADMIN`：拥有全部管理员权限
- `KNOWLEDGE_READ`：允许 GET、HEAD
- `KNOWLEDGE_WRITE`：允许 POST、PUT、PATCH、DELETE

为兼容旧环境，仍支持 `ADMIN_API_TOKEN` 作为单个 `ADMIN` 账户的回退配置，但新环境应优先使用 `ADMIN_ACCOUNTS`。

前端开发环境使用账户 token 配置 `VITE_ADMIN_API_TOKEN`。鉴权行为如下：

| 场景 | HTTP 状态码 | 说明 |
| --- | ---: | --- |
| 令牌正确 | `2xx` | 正常执行接口 |
| 令牌缺失或错误 | `401` | 管理员身份校验失败 |
| 后端未配置令牌 | `503` | 服务端鉴权配置缺失 |
| `OPTIONS` 预检 | `200` | 允许跨域预检通过 |

管理员接口的访问结果会写入后端审计日志，日志只记录请求方法、路径、状态码和来源地址，不记录令牌原文。

## 1. 健康检查接口

用于验证后端服务是否正常运行。

### 请求信息

```http
GET /api/health
```

完整地址：

```text
http://127.0.0.1:8080/api/health
```

请求参数：无

### 成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "status": "ok"
  }
}
```

### 响应字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `data.status` | `string` | 后端服务状态，`ok` 表示服务正常。 |

### 验证命令

```powershell
Invoke-RestMethod 'http://127.0.0.1:8080/api/health' | ConvertTo-Json -Depth 8
```

## 2. 数据库连通性检查接口

用于验证后端是否可以正常连接 MySQL，并返回数据库当前时间。

### 请求信息

```http
GET /api/db/ping
```

完整地址：

```text
http://127.0.0.1:8080/api/db/ping
```

请求参数：无

### 成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "timestamp": "2026-07-20 09:23:43",
    "status": "ok"
  }
}
```

### 成功响应字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `data.timestamp` | `string` | MySQL 当前时间，格式为 `yyyy-MM-dd HH:mm:ss`。 |
| `data.status` | `string` | 数据库连通状态，`ok` 表示连接正常。 |

### 失败响应

```json
{
  "code": 500,
  "message": "Database ping failed: connection timeout",
  "data": {
    "timestamp": null,
    "status": "error"
  }
}
```

### 失败响应字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `code` | `number` | 失败时当前返回 `500`。 |
| `message` | `string` | 数据库连接失败原因。 |
| `data.timestamp` | `null` | 连接失败时没有数据库时间。 |
| `data.status` | `string` | `error` 表示数据库连通性检查失败。 |

### 验证命令

```powershell
Invoke-RestMethod 'http://127.0.0.1:8080/api/db/ping' | ConvertTo-Json -Depth 8
```

## 3. 知识库文档模块

本模块用于管理知识库文档元数据。本阶段只处理 `kb_document` 表中的元数据，不包含文件上传、文档解析、文本切分、向量化或 Elasticsearch 接入。

基础路径：

```text
/api/admin/knowledge/documents
```

### 3.1 新增文档元数据

```http
POST /api/admin/knowledge/documents
```

请求示例：

```json
{
  "title": "中华人民共和国劳动合同法",
  "document_type": "LAW",
  "issuing_authority": "全国人民代表大会常务委员会",
  "canonical_source_url": "https://example.test/labor-contract-law",
  "jurisdiction_code": "CN",
  "scope_text": "全国范围",
  "authority_level": 100,
  "status": "DRAFT"
}
```

必填字段：

| 字段 | 说明 |
| --- | --- |
| `title` | 文档标题。 |
| `document_type` | 文档类型，例如 `LAW`、`POLICY`、`FAQ`。 |

默认值：

| 字段 | 默认值 |
| --- | --- |
| `jurisdiction_code` | `CN` |
| `authority_level` | `100` |
| `status` | `DRAFT` |
| `lock_version` | `0` |

成功响应示例：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "public_id": "ABCDEF1234567890ABCDEF1234",
    "title": "中华人民共和国劳动合同法",
    "document_type": "LAW",
    "issuing_authority": "全国人民代表大会常务委员会",
    "canonical_source_url": "https://example.test/labor-contract-law",
    "jurisdiction_code": "CN",
    "scope_text": "全国范围",
    "authority_level": 100,
    "current_version_id": null,
    "status": "DRAFT",
    "created_by": null,
    "lock_version": 0,
    "created_at": "2026-07-20T13:00:00",
    "updated_at": "2026-07-20T13:00:00"
  }
}
```

校验失败响应示例：

```json
{
  "code": 400,
  "message": "title is required",
  "data": null
}
```

PowerShell 验证命令：

```powershell
$body = @{
  title = '中华人民共和国劳动合同法'
  document_type = 'LAW'
  issuing_authority = '全国人民代表大会常务委员会'
  jurisdiction_code = 'CN'
  status = 'DRAFT'
} | ConvertTo-Json -Depth 8

Invoke-RestMethod 'http://127.0.0.1:8080/api/admin/knowledge/documents' -Method Post -ContentType 'application/json' -Body $body | ConvertTo-Json -Depth 8
```

### 3.2 分页查询文档列表

```http
GET /api/admin/knowledge/documents
```

查询参数：

| 参数 | 说明 |
| --- | --- |
| `title` | 按标题模糊查询。 |
| `document_type` | 按文档类型精确查询。 |
| `status` | 按状态精确查询。 |
| `issuing_authority` | 按发布机构模糊查询。 |
| `jurisdiction_code` | 按辖区代码精确查询。 |
| `pageNo` | 页码，默认 `1`。 |
| `pageSize` | 每页条数，默认 `10`，最大 `100`。 |

成功响应示例：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "records": [
      {
        "id": 1,
        "public_id": "ABCDEF1234567890ABCDEF1234",
        "title": "中华人民共和国劳动合同法",
        "document_type": "LAW",
        "status": "DRAFT"
      }
    ],
    "total": 1,
    "pageNo": 1,
    "pageSize": 10,
    "pages": 1
  }
}
```

PowerShell 验证命令：

```powershell
Invoke-RestMethod 'http://127.0.0.1:8080/api/admin/knowledge/documents?title=劳动合同&document_type=LAW&pageNo=1&pageSize=10' | ConvertTo-Json -Depth 8
```

### 3.4 更新文档元数据

```http
PUT /api/admin/knowledge/documents/{id}
```

支持更新字段：

```text
title, document_type, issuing_authority, canonical_source_url, jurisdiction_code, scope_text, authority_level, status
```

不会更新：

```text
public_id, created_at
```

请求示例：

```json
{
  "title": "中华人民共和国劳动合同法（修订版）",
  "document_type": "LAW",
  "issuing_authority": "全国人民代表大会常务委员会",
  "canonical_source_url": "https://example.test/labor-contract-law",
  "jurisdiction_code": "CN",
  "scope_text": "全国范围",
  "authority_level": 100,
  "status": "PUBLISHED"
}
```

成功响应示例：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "title": "中华人民共和国劳动合同法（修订版）",
    "document_type": "LAW",
    "status": "PUBLISHED"
  }
}
```

数据不存在响应示例：

```json
{
  "code": 404,
  "message": "Knowledge document not found",
  "data": null
}
```

PowerShell 验证命令：

```powershell
$body = @{
  title = '中华人民共和国劳动合同法（修订版）'
  document_type = 'LAW'
  issuing_authority = '全国人民代表大会常务委员会'
  jurisdiction_code = 'CN'
  status = 'PUBLISHED'
} | ConvertTo-Json -Depth 8

Invoke-RestMethod 'http://127.0.0.1:8080/api/admin/knowledge/documents/1' -Method Put -ContentType 'application/json' -Body $body | ConvertTo-Json -Depth 8
```

```

```
